<a name="description_task"><h3>Описание задания:</h3></a>

Банк АО "Рога и Копыта" открыл новую услугу – брокерский счёт.
Нужно сделать интеграцию формы с сайта с внутренними ресурсами организации через **REST API**.

Ориентировачная нагруженность формы **~20 человек в час**.

Форма состоит из трех шагов:
* Шаг 1 – Заполнение паспортных данных и суммы депозита
* Шаг 2 – Выбор типа счёта
* Шаг 3 – Подписание договора

---

<a name="algo_service"><h3>Алгоритм работы интеграции:</h3></a>

* Шаг 1: Сайт отправляет запрос с данными в сервис –≥ сервис проверяет и валидирует данные, *конвертирует валюту в рубли по актуальному курсу, записывает данные в базу и отдает ответ в формате JSON.
* Шаг 2: Сайт отправляет запрос с данными в сервис –≥ сервис проверяет и валидирует данные, записывает их в базу, генерирует документ (файл dogovor.docx), отдаёт сайту в ответе документ в формате PDF(ответ в формате JSON). Также сервис сохраняет обе версии документа в файловой структуре.
* Шаг 3: Сайт отправляет запрос с данными в сервис –≥ сервис проверяет данные и валидирует их, записывает в базу.

<a name="task_project"><h3>Задачи проекта:</h3></a>
1. Исходя из условий задания подобрать оптимальный стек технологий;
2. На основаниии своего опыта продумать алгоритм работы сервиса и собрать архитектуру;
3. Собрать боевой стенд в docker для последующего деплоя на сервер.

В качестве базы можно взять sqlite. 
>Таблицы в бд: 
>1. client (данные клиента)
>2. dogType(данные типа выбранного договора)
>3. sms(данные по смс)

Актуальный курс валют брать с сайта https://www.cbr.ru/currency_base/daily/

В файле dogovor.docx желтым цветом проставлены заменяемые данные.

##### Ограничения:
* Python >= 3.10;


##### Основные задачи:
* Клонировать данный репозиторий. Создать ветку dev и вносить изменения в неё.
* Собрать и продумать архитектуру проекта и методов API;
* Реализовать каждый метод;
* Валидация данных по формату;
* Отлов ошибок и сбоев;
* Соблюдать основные правила PEP8;
* Соблюдение форматов ответов;
* Файлы документов должны быть доступны вне контейнера.

##### Будет плюсом:
* Учесть в коде все возможные "подводные камни" и ошибки пользователя;
* docstrings;
* Логирование;
* Файлы логов должны быть доступны вне контейнера.

<a name="dop_task"><h3>Дополнительная задача</h3></a>

**Дополнительная усложненная задача (не обязательная, но будет плюсом).**

Напишите скрипт, который будет выводить 4 даты:
1. Первая дата сегодняшний день, если он рабочий. Если это выходной день, то ничего не выводится.
2. Остальные три даты это +3 рабочих дня к текущей дате.

Рабочими днями являются официальные рабочие дни по производственному календарю РФ. 

Три примера:
Допустим сегодня 22.02.2024, тогда скрипт должен вывести:
```
22.01.2024 
23.01.2024
24.01.2024
25.01.2024
```

Допустим сегодня 25.01.2024:
```
25.01.2024
26.01.2024
29.01.2024
30.01.2024
```

Допустим сегодня 20.01.2024:
```
22.01.2024
23.01.2024
24.01.2024
```
